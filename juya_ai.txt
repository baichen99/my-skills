<h2 id="zhi-pu-xiang-glm-coding-plan-yong-hu-kai-fang-glm-5-1-mo-xing-shi-yong-1"><a href="https://mp.weixin.qq.com/s/5g5-cJSuQumzZDVgiCaTuQ">智谱向 GLM Coding Plan 用户开放 GLM-5.1 模型使用</a> <code>#1</code></h2>
<blockquote>
<p>智谱向 <strong>GLM Coding Plan</strong> 全体用户开放了未正式发布的 <strong>GLM-5.1</strong> 模型的使用，该模型定位为能处理长周期任务的新一代编程 Agent。根据官方发布的基准测试数据，该模型较前代 <strong>GLM-5</strong> 模型有显著提升。</p>
<p>用户需在 Coding Agent 配置中手动将模型名称更改为 <code>glm-5.1</code> 方可接入使用。</p>
<p>官方称计划于 <strong>4月上旬</strong> 正式发布该模型。</p>
</blockquote>
<p>智谱向 <strong>GLM Coding Plan</strong> 全部层级（Lite/Pro/Max）用户超前开放了尚未正式发布的 <code>glm-5.1</code> 模型。用户需在所使用的 Coding Agent 自定义配置中手动将模型名称更改为“glm-5.1”进行接入。</p>
<p>该模型作为高阶模型采用差异化额度消耗机制，高峰期（每日 <strong>14:00-18:00 UTC+8</strong>）按 <strong>3倍</strong> 系数抵扣，非高峰期常规按 <strong>2倍</strong> 抵扣。但作为限时福利，非高峰期抵扣系数降至 <strong>1倍</strong> 并持续至 <strong>4月底</strong>。</p>
<p>官方提示，在高负载时可能触发保护性限流，并将优先保障编程任务请求。<strong>OpenClaw</strong> 等非核心编程场景在集群繁忙时更易遇到限流。</p>

<h2 id="kwaikat-fa-bu-kat-coder-pro-v2-mo-xing-2"><a href="https://mp.weixin.qq.com/s/5JpX2UxW-SKpyPR4yWOVEw">KwaiKAT 发布 KAT-Coder-Pro V2 模型</a> <code>#2</code></h2>
<blockquote>
<p>快手KwaiKAT团队发布了编程模型 <strong>KAT-Coder-Pro V2</strong>。该模型有效攻克了工具调用失败与长链路任务中断等痛点，已上线官方 <code>API</code> 平台和 <code>Coding Plan</code> 套餐。</p>
</blockquote>
<p>快手 KwaiKAT 正式发布旗舰级 Agentic Coding 模型 <strong>KAT-Coder-Pro V2</strong>。该模型在 Agentic 场景下具备脚手架泛化能力，兼容十多种主流 AI 编码工具，并针对 <code>OpenClaw</code> 进行了专项训练与深度优化，以解决工具调用失败及长链路任务中断等问题。</p>
<p>同时，官方称该模型在前端美学生成方向取得突破，在 Landing Page 和 PPT 场景中支持通过口语化描述生成高品质输出。</p>
<p>目前，<strong>KAT-Coder-Pro V2</strong> 已全量上线，用户可通过 <code>StreamLake.com</code> 平台进行 API 调用或订阅 <strong>Coding Plan</strong> 套餐使用。</p>

<h2 id="meta-fa-bu-sam-3-1-zhi-chi-dan-ci-zhui-zong-16-ge-mu-biao-3"><a href="https://go.meta.me/8dd321">Meta 发布 SAM 3.1，支持单次追踪 16 个目标</a> <code>#3</code></h2>
<blockquote>
<p>Meta 推出了 <strong>SAM 3.1</strong> 模型，引入 <code>对象多路复用</code> 技术，实现了单次计算同时追踪最多 <strong>16</strong> 个视频目标，大幅消除算力浪费。</p>
</blockquote>
<p>Meta 官方发布了 <strong>SAM 3.1</strong> 模型，作为 <strong>SAM 3</strong> 的直接替换版本。该模型核心引入“对象多路复用”技术，官方称其能在不牺牲准确率前提下，于单次计算中同时追踪多达 <strong>16</strong> 个目标。相较于此前需单独计算的模式，该改进消除了算力浪费，显著提升视频处理效率。</p>
<p><img src="https://cdn.jsdelivr.net/gh/imjuya/picx-images-hosting@master/imagehub/20260328/20260328081319_84cecdf28b.png" alt="" /></p>

<h2 id="claude-code-shang-xian-windows-powershell-yuan-sheng-yun-xing-zhi-chi-4"><a href="https://code.claude.com/docs/en/tools-reference#powershell-tool">Claude Code 上线 Windows PowerShell 原生运行支持</a> <code>#4</code></h2>
<blockquote>
<p><strong>Claude Code</strong> 针对 Windows 平台推出了更新，支持原生运行 <code>PowerShell</code> 命令以替代原有的 <code>Git Bash</code> 路由。目前该功能暂不支持自动模式与沙盒机制。</p>
</blockquote>
<p><strong>Claude Code</strong> 近期面向 Windows 平台推出了一项需主动启用的预览版 <code>PowerShell</code> 工具，允许其原生运行 <code>PowerShell</code> 命令而无需经由 <code>Git Bash</code> 路由。</p>
<p>用户可通过配置环境变量或 <code>settings.json</code> 文件来启用该功能。启用后，系统会自动检测 <code>pwsh.exe</code> (<code>PowerShell</code> 7+) 并向下回退兼容 <code>powershell.exe</code> (<code>PowerShell</code> 5.1)。</p>
<p>同时，官方提供了在交互命令默认设置、命令挂钩及技能前言中指定使用 <code>PowerShell</code> 的相关配置项。</p>
<p>由于处于预览阶段，该工具目前暂不支持 <code>Auto</code> 模式、配置文件加载、沙盒机制以及 <code>WSL</code> 环境，且启动 <strong>Claude Code</strong> 仍需依赖 <code>Git Bash</code>。</p>
<p><img src="https://cdn.jsdelivr.net/gh/imjuya/picx-images-hosting@master/imagehub/20260328/20260328075244_646a4fec3a.png" alt="" /></p>

<hr />
<h2 id="openrouter-xian-zhi-duo-di-yong-hu-diao-yong-bu-fen-mo-xing-cheng-zun-shang-you-yao-qiu-5"><a href="https://www.landiannews.com/archives/112371.html">OpenRouter 限制多地用户调用部分模型，称遵上游要求</a> <code>#5</code></h2>
<blockquote>
<p>据报道，<strong>OpenRouter</strong> 目前已限制中国大陆等地区的用户调用部分特定模型，触发该限制时会报错 <code>403 Author Banned</code>。<strong>OpenRouter</strong> 称该举措系遵守上游提供商要求。</p>
</blockquote>
<p>据报道，<strong>OpenRouter</strong> 近期阻止了中国大陆等地区的用户使用其平台上的部分模型。该平台针对性地限制了特定模型的调用权限，当受影响用户尝试调用这些模型时，系统会提示“403 Author Banned”等信息。</p>
<p><strong>OpenRouter</strong> 解释称，采取上述政策是为了遵守上游模型提供商的要求。</p>
<p><img src="https://cdn.jsdelivr.net/gh/imjuya/picx-images-hosting@master/imagehub/20260328/20260328082107_499cb779bf.png" alt="" /></p>

<blockquote>
<p>Anthropic 因外部工具配置错误泄露了代号为“Capybara”的未发布新模型信息，该模型正式名称将为 <strong>Claude Mythos</strong>，定位在 <strong>Claude Opus</strong> 之上的全新第四档，在编码、推理及网络安全能力上实现了跨越式提升。</p>
<p>鉴于其极高的网络攻击风险，<strong>Anthropic</strong> 采取了罕见的防御优先策略，目前仅向网络安全防御机构开放早期访问以建立防守优势，暂不面向公众。</p>
<p>同时，因运行成本极高，该模型需大幅优化效率后才会考虑广泛发布。</p>
</blockquote>
<p>Anthropic 因外部 CMS 配置错误，意外泄露未发布模型 <code>Claude Mythos</code>（代号 Capybara）信息，涉及近 <strong>3000</strong> 份早期草稿。该事件由安全研究员发现并经媒体核实。</p>
<p>官方确认该模型为 Claude Opus 之上的全新第四档，在编码及网络安全测试中得分大幅超越 <code>Claude Opus 4.6</code>。</p>
<p>鉴于官方警告其网络能力“远超任何其他 AI 模型”，存在引发漏洞利用浪潮风险，该公司优先向网络安全防御机构开放早期访问。</p>
<p>官方坦言该模型运行成本极高，需优化效率后才考虑广泛发布。</p>
