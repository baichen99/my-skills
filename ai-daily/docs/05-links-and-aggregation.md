# 原文链接与聚合页（强制）

日报每条新闻末尾 `[原文](…)` 中的 URL **必须是可追溯的一手来源**（外站文章、官方公告、仓库、论文页等），**禁止**把下列链接当作「原文」敷衍了事：

- 聚合站的**列表页、频道首页、日报封面页**
- 仅指向聚合站域名下的**站内详情/话题页**（若该页不是原文本身）

**配置不是万能**：Recipe/YAML 只能覆盖「页面上稳定出现、可选中」的字段；**很多聚合源根本不展示外站 URL，或只在详情异步出现**，因此**不能指望单靠加配置就解决全部链接**——必须与 [02-workflow.md](02-workflow.md) 中的「补链与校验阶段」配合。

## 按来源类型处理（首轮 + 补链）

- **RSS/Atom**：以每条 `<item>` 的 `<link>`（或等价的 permalink `guid`）为原文 URL；不要用 feed 页或站点首页。若 `<link>` 指向聚合站内页，走补链阶段进详情解析。
- **HTML 列表（Readhub、类日报聚合等）**：
  - 首轮：优先从列表项上指向**外域**的 `href`、`data-source-url`、或「来源」行上的外链抽取
  - 若只有站内路径或**无外链**：进入 **补链阶段**——打开详情页（或跟随 302），取 **落地后的外站 URL**，或页面内「原文」「来源」的链接
  - 可辅助：JSON-LD、`og:url`（当与聚合域名不一致时）、正文首个外域引用
- **去重**：对同一事件优先按 **规范化后的原文 URL** 去重，避免同一新闻因不同聚合页链接重复出现

## Recipe 约定

聚合类站点应在 YAML 中尽量配置能拿到 **title**、**入口 url** 以及**若存在则**抽取 `sourceUrl` 的选择器；验证 recipe 时至少抽查 3 条。**未配置到 `sourceUrl` 不等于失败**，只要在补链阶段能补齐即可。
