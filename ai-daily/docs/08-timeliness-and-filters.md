# 时效与「今日」过滤（强制）

日报文件名中的日期（如 `ai-daily-frontend-2026-03-30.md`）表示**内容应对齐该日或用户约定的时间窗口**。若成稿里混入**非窗口内**发布的旧闻，属于**过滤失败**，必须在抓取与分析阶段拦截。

## 时间窗口（默认）

- 用户未特别说明时：**最近 24 小时**，或**日历日 `YYYY-MM-DD`（按东八区理解「当天」）**，二者在技能执行前与用户约定其一。
- 所有「今日资讯」条目，必须能在**原文或元数据**上对应到该窗口。

## 每条收录前必须核验（至少其一）

| 来源类型 | 核验方式 |
|----------|----------|
| RSS/Atom | `<pubDate>` / `<updated>` / `dc:date`；**按日期排序**后只取窗口内条目，勿只取列表前 N 条 |
| 文章页 | `article:published_time`、`datePublished`（JSON-LD）、正文「发表于 / 发布于」可见日期 |
| GitHub | Release / Tag 日期、commit 日期（Trending 仅表示「当前热」，**不等于今日新事**，需点进仓库看最近活动是否落在窗口内） |
| 掘金/社区列表 | 列表常混 **热门推荐与发布时间**；必须以卡片/详情上的**发布时间**为准，不能只看排序 |

> 浏览器使用补充（强制）：当来源提供的发布时间属于“元数据但缺少可见发布日期”（例如 RSS 只有模糊字段、或聚合页未直接展示发布时间）时，必须打开原文/详情页（Agent Browser）定位页面可见的发布时间后再确认“今日窗口”。

## 必须剔除或单独标注的情况

- 原文 **发布日期早于窗口**（例如要求「今日」却收录三天前文章），除非用户明确要求「本周精选」等另一口径。
- **仅有「热门」排序、无可靠发布时间**且无法打开原文确认时：**不收录**，或列入「未核实日期」并说明，**不**放入主「今日资讯」。
- 聚合站把**旧闻**重新顶到列表：以**原文首次发布日期**为准，不以「抓取当日出现在首页」为准。

## 分析阶段收口

- 成稿前做一次 **「日期抽查」**：对每条「今日资讯」自问：原文日期是否在窗口内？若否，删除或改写口径（改为「近期值得读」并改标题/文件名日期）。
- 「今日重点」里的「条目总数」须与**通过日期过滤后的条数**一致。

## 与补链的关系

补链阶段拿到原文 URL 后，应顺带读取**页面可见发布日期**；若发现与「今日」不符，**从成稿中移除该条**并记入「排除清单：原因=日期不符」。
